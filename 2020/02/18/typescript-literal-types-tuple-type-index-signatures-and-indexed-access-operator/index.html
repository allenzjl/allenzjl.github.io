观察以下代码，定义了接口 A 包含成员 field，跟随在类型 A 后面的中括号即为 索引访问操作符， 返回的是 field 的类型 string

同时 Typescript 允许在接口和类的定义中声明 索引签名，比如

[field: string]: string 指接口 B 下可以有任意个类型为 string 的索引（索引类型只能为 string 或 number，前面的 field 只是提高可读性并无任何意义），这些索引对应的成员类型为 string | number 。下面的代码通过 typeof 操作符捕获对应字段的类型，因为声明了 string 类型的索引签名，所以可以通过后面的索引访问操作符捕获 string 索引签名的类型 string | number 。要注意的是如果没有声明 string 索引签名是没法通过 Type[string] 获取类型的

回到最开头的代码，typeof ButtonTypes[number] 对元组能执行索引访问操作 [number] 原因很可能就在于元组内部定义了 [x: number]: Type，其中 Type 表示元组定义时包含的所有联合类型。因为 Typescript 元组本身并没有内部类型，而是直接翻译成 Javascript 的数组，通过查看 lib.es5.d.ts 的定义，在 interface Array<T> 下确实找到了 [n: number]: T; ，猜想得到证实。用代码模拟如下：
